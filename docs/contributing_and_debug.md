# Contributing and Debugging Docs
> the following information is subject to change as the project evolves
> as of right now there is no contributor code of conduct aside from what is stated in our AI Usage Policy in the README.md


## The Basics of Contributing
Contributing can be done through the following steps:
1. Fork the repository on GitHub.
2. Clone the forked repository to your local machine.
3. Create a new branch for your changes.
4. Make your changes and commit them.
5. Push your changes to your forked repository.
6. Create a pull request on GitHub.

## Debugging
when debugging you can use the following commands:
```.\run.bat``` can be used to start the application in debug mode. If you happen to be working on the project on a Unix or POSIX system, you can use the following command:
```./run.sh``` can be used to start the application in debug mode.

## Building & Compiling Core Modules
This project uses [Cython](https://cython.org/#download) for its core modules, to compile them please refer to [building.md](./building.md) to get started.

## Important Reminders
this repo does use the UV package manager for python, please use UV for adding packages or running the project
if you are unfamiliar with UV or how to install it you can refer to the following site to get started with [uv](https://docs.astral.sh/uv/)
